# Top.AlphaPulse 附录C 验收用例清单

## 1. 文档说明

本文档用于补充主需求文档中的验收标准，提供面向产品、研发和测试的可执行验收用例清单。用例按模块与横向能力组织，并区分当前阶段可验收项与后续阶段预留项。

## 2. 验收范围说明

### 2.1 当前阶段重点验收范围

- 页面可访问性。
- 数据加载与降级表现。
- 导航、搜索和跨模块跳转。
- 核心模块的基础数据展示和交互。

### 2.2 后续阶段预留验收范围

- 真实 AI 分析接口。
- 智能选股策略保存、回测和预警。
- 个股详情高级数据面板。
- 统一资讯模型下的公告、研报和财经日历能力。

## 12. 验收执行建议

- 产品验收优先关注业务路径完整性和交付边界是否清晰。
- 研发验收优先关注数据结构、接口约束、异常场景和性能表现。
- 测试验收需覆盖有数据、无数据、慢数据、错误数据和降级场景。
- 对“后续”阶段用例单独建立里程碑，不应与当前交付验收混用。
